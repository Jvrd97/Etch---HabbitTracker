# ADR-0001: Переиспользуем существующий FastAPI-бэкенд

## Status

Accepted (грилл 2026-07-21).

## Context

Есть работающий FastAPI + PostgreSQL бэкенд с гибкой моделью (категории → кастомные поля → записи, журнал), покрытый 49 тестами. Нужен API для нативного iOS-приложения. Альтернативы: переписать с нуля по всем стандартам проекта или гибрид.

## Decision

Переиспользуем бэкенд как есть. Доработки точечные: endpoint табличной агрегации, API-key dependency, прод-профиль compose. Стандарты проекта (uv, mypy --strict, ruff) подтягиваем инкрементально отдельным quick-win, не блокируя фичи.

## Consequences

- (+) iOS-приложение можно начинать сразу, контракт API уже есть.
- (+) 49 тестов защищают от регрессий при доработках.
- (−) В коде временно сосуществуют старый стиль (pip/requirements.txt) и целевой (uv); закрывается отдельным issue.
