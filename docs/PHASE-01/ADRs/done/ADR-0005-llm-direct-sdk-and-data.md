# ADR-0005: LLM-слой на голом anthropic-sdk; личные данные уходят в Anthropic API

## Status

Accepted (грилл 2026-07-22).

## Context

Правила проекта требуют изолированный LLM orchestration-слой и упоминают LangChain. Первый юзкейс — «инсайты по кнопке»: один провайдер, один промпт. Для разбора нужны и метрики, и тексты журнала — это личные данные о здоровье и быте пользователя.

## Decision

1. Слой `app/llm/` изолирован (бизнес-сервисы LLM не зовут — правило сохраняется), но реализуется на голом anthropic-sdk без LangChain: фреймворк добавим, когда появится второй юзкейс (голосовой ввод #19) и реальная потребность в абстракции.
2. В Anthropic API осознанно отправляются агрегированные метрики и полные тексты журнала за период. Приложение single-user, пользователь — владелец данных, решение принято им в грилле. В логи приложения контент запросов/ответов и ключ не пишутся.
3. Модель — claude-sonnet-5; ключ — `ANTHROPIC_API_KEY` в env сервера; пустой ключ = фича выключена (503), не падение.

## Consequences

- (+) Минимум зависимостей и магии; промпт и клиент читаются целиком.
- (+) Приватность управляема: одна точка выхода данных, легко аудитить.
- (−) При втором провайдере/юзкейсе придётся вводить абстракцию — осознанный долг.
- (−) Тексты журнала покидают сервер — при появлении чужих пользователей решение нужно пересмотреть (opt-in).
