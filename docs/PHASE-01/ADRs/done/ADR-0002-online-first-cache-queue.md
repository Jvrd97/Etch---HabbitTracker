# ADR-0002: Online-first с read-кэшем и очередью записей

## Status

Accepted (грилл 2026-07-21).

## Context

iPhone ходит на сервер через Tailscale; связь может отсутствовать (сервер спит, VPN отвалился, нет сети в зале). Полный offline-first (локальная БД как источник истины + двусторонняя синхронизация + конфликт-резолюшен) — самая дорогая часть мобильной разработки и для одного пользователя не окупается.

## Decision

Online-first: сервер — источник истины. На устройстве SwiftData хранит (а) кэш последних успешных ответов для чтения без сети и (б) очередь исходящих мутаций (создание записей), которая автоматически флашится при появлении связи (NWPathMonitor + retry).

## Consequences

- (+) Таблица и история видны всегда; запись в зале без сети не теряется.
- (+) Нет конфликт-резолюшена: мутации применяются в порядке очереди, конфликтовать некому (один пользователь).
- (−) Редактирование/удаление без сети не гарантируем в MVP — только создание встаёт в очередь.
- (−) Кэш может отставать от сервера до следующего успешного запроса — приемлемо.
