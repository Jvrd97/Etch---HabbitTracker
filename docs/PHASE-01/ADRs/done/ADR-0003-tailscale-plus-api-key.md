# ADR-0003: Периметр Tailscale + статический API-ключ

## Status

Accepted (грилл 2026-07-21).

## Context

Бэкенд крутится на VPS, клиент — личный iPhone. Полноценная аутентификация (JWT, юзеры) избыточна для одного пользователя, но полагаться только на сеть — ноль страховки от случайно открытого порта или чужого устройства в tailnet.

## Decision

Два рубежа: (1) сервис доступен только внутри tailnet, порты наружу не публикуются; (2) все endpoints требуют заголовок `X-API-Key` со статическим ключом. Ключ живёт в env на сервере (`.env`, не в git) и в Keychain на iPhone. Мультиюзерность — out of scope фазы.

## Consequences

- (+) Почти ноль кода: одна FastAPI dependency.
- (+) Утечка порта наружу не превращается в открытую БД привычек.
- (−) Ротация ключа ручная (поменять env + перезапуск + обновить в настройках приложения).
- (−) Тесты должны прокидывать ключ (фикстура в conftest).
