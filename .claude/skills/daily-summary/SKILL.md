# Daily Summary — из голосового дампа в трекер

## Что делает

1. **Парсит** сырую расшифровку (STT-шум, повторы, оговорки — нормализовать, смысл не менять).
2. **Пишет в трекер** через API (`http://localhost:8000/api/v1`, dev — без ключа; прод — `X-API-Key`):
   - числовые метрики → `POST /entries/` в подходящую категорию (Push-ups, Running Outdoor, …). Сопоставлять по смыслу: «отжался 20» → Push-ups: 20; «побегал 28 км» → Running Outdoor: 28.
   - витамины/чек-листы → `PUT /entries/checklist`.
   - весь день → `POST /journal/`: title = краткая тема дня, content = MD-саммари (шаблон ниже), mood из тона рассказа, tags по темам (health, sport, work, food, travel…).
3. **Показывает** итоговое саммари в чате + флаги.

## Шаблон MD-саммари (content журнала)

```markdown
## День коротко
<2-3 предложения>

## Спорт и тело
## Работа
## Питание
## Хорошее / Благодарность
## Плохое / Отметить
## Завтра
<явные намерения из рассказа>
```

## Правила

- **Не выдумывать**: только то, что сказано. Неправдоподобные числа (28 км больным?) записывать как сказано, но флагом спросить пользователя.
- **Никаких медицинских советов** сверх того, что пользователь сам отметил.
- Категории/поля брать из `GET /categories/` по факту; если метрике нет категории — предложить создать, не создавать молча.
- Даты: локальный день пользователя; если рассказ про «вчера» — уточнить дату.
- Ответ в чате: короткое саммари + список записанного в трекер + флаги. Полный MD — в journal, не дублировать в чат целиком.
