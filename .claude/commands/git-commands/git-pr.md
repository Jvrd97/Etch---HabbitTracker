---
description: "Generate a PR description from the current branch's commits and closed issues."
---

Generate a PR description for the current branch.

Process:
1. Determine base branch:
   - Default: `main` (or `master` if main doesn't exist)
   - If $ARGUMENTS: use it as base

2. Gather data:
   - Branch name: `git branch --show-current`
   - Commits: `git log <base>..HEAD --oneline`
   - Diff stats: `git diff <base>..HEAD --stat`
   - Issues closed (look at `issues/closed/` files modified or moved in this branch):
     ```bash
     git log <base>..HEAD --diff-filter=A --name-only -- issues/closed/
     ```

3. Format PR description:
   ```markdown
   ## Summary
   <2-3 sentences describing what this PR does>

   ## Changes
   <Bullet list of significant changes, grouped by area>
   - **<area>**: <what changed and why>
   - ...

   ## Testing
   <How was this verified>
   - Unit tests: <added/modified, count>
   - Integration tests: <added/modified, count>
   - Manual QA: <if relevant>

   ## Closes
   - Closes #<issue-1>
   - Closes #<issue-2>
   - ...

   ## Out of Scope
   <Anything explicitly NOT addressed in this PR but related>

   ## Reviewer Notes
   <Optional: anything the reviewer should pay extra attention to>
   ```

4. Save to `/tmp/pr-description-<branch>.md` and print path. Ask user:
   - `[c]opy to clipboard` (run `pbcopy < /tmp/...`)
   - `[g]h create PR` (run `gh pr create --body-file /tmp/...`)
   - `[s]how me, I'll handle it`

Rules:
- Never invent test runs that didn't happen — only describe what was added
- "Reviewer Notes" only if there's a real risk worth flagging (security, perf, breaking)
- Keep "Summary" tight — reviewer reads this first
- Don't include "Generated by Claude" or similar attribution
